# IntegraRP v1.20 — task-unification

Base: `fc6841358734576b4a32f4e174d4e2f3045d68a1`.

Este documento registra a evolução incremental realizada nesta branch para schema explícito `integrarp`, geração do `script_completop.sql`, validação canônica e limitações verificadas no ambiente.

## Estado implementado

- Migrations inválidas conhecidas tiveram blocos órfãos de registro manual removidos.
- O uso de `search_path` identificado na migration 0023 foi removido.
- A migration 0027 deixou de criar dependência ativa em `integrarp.workflow_task` e passou a evoluir `integrarp.tarefa_operacional` como fila canônica.
- O validador usa nomes canônicos em português para Flow.
- O gerador canônico não emite `SET search_path` e gera `script_completop.sql` e `scriptcompleto.sql` com o mesmo conteúdo.

## Evidências e limitações

- Validação .NET completa não foi executada porque o comando `dotnet` não está instalado no ambiente.
- Validação PostgreSQL com `psql` não foi executada porque não há instância PostgreSQL 16 provisionada neste ambiente.
- Standalone Windows real não foi executado porque o ambiente é Linux.
- Os critérios não comprovados devem manter o PR em Draft.
