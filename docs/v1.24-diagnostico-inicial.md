## Erros encontrados e causas

1. SDK .NET ausente no runner, impedindo restore/build/test/format.
2. Clone sem branch `main` local/remota; a base foi validada por SHA e ancestralidade na HEAD disponível.
3. Login Web chamava `/auth/login`, usava payload incompatível e adicionava role fixa `Administrador`.
4. Pipeline da API registrava `GlobalExceptionMiddleware` tarde demais para capturar falhas de middlewares anteriores.
5. Migration 0029 introduziu contrato paralelo de lockout; v1.24 adiciona reconciliação aditiva em 0030.

## Resultado posterior implementado

- Login Web passou a chamar `/api/auth/login` com `tenantSlug`, `email`, `password`, `deviceId` e `deviceName`.
- Role fixa foi removida; perfis/permissões vêm da resposta real da API.
- Tokens são armazenados server-side protegidos por Data Protection.
- Logout Web passou a ser somente POST com antiforgery, revogação API e remoção do ticket/token store.
- Migration 0030 foi criada para reconciliar lockout, histórico de senha e dashboard operacional.
- `script_completop.sql` e `scriptcompleto.sql` foram regenerados para v1.24.
