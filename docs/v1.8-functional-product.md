# v1.8 — Produto funcional completo

A v1.8 consolida o IntegraRP como produto operacional: telas principais devem orientar o usuário, CRUDs devem persistir dados reais, filtros devem ter paginação e ações devem apontar para APIs reais.

## Escopo entregue neste incremento

- Fonte de banco atualizada com objetos v1.8.
- Catálogo inicial de templates funcionais.
- Modelo persistente para auditoria tela a tela.
- Modelo persistente para validação por módulos.
- Documentação operacional para produto, CRUD, jornadas, layout, API, worker e mobile.

## Diretriz

Nenhuma tela deve depender de dado fixo como fonte principal. Estados vazios são permitidos apenas quando acompanhados de CTA e próxima ação.
