# V1.8 — Real functional product

V1.8 introduces real domain persistence as the baseline for operational functionality. The database now includes domain-specific tables for admin, commercial, inventory, orders, tasks, billing, outbox, journey, audit and operational templates.

## Implemented in this change

- Idempotent migration `0020_v18_funcionalidade_real_produto.sql`.
- `database/scriptcompleto.sql` updated with the same domain model.
- PostgreSQL repository classes separated by domain and pointing to real tables instead of `integrarp.v15_operational_object`.

## Acceptance stance

This is a functional foundation slice. API/use-case/web endpoints must consume these repositories and tables in follow-up increments before all requested V1.8 vertical slices can be considered complete.
