# IntegraRP v1.9 — demo funcional

## Como rodar e validar

1. Configure `ConnectionStrings__IntegraRP` ou informe a connection string no script.
2. Execute `scripts/db-debug-scriptcompleto.ps1` para criar `integrarp_debug`, aplicar `database/scriptcompleto.sql`, reaplicar o script e executar `database/validation/validate_scriptcompleto_v19.sql`.
3. Se `psql` não estiver disponível localmente, use o workflow `database-scriptcompleto-v19` no GitHub Actions.

## O que a versão demonstra

A seed v1.9 carrega tenant `demo`, usuários por perfil, permissões, setores, clientes, contatos, endereços, produtos, categorias, estoque, pedidos, itens, tarefas, fatura, títulos, boleto fake, outbox, jornada, templates e atividades operacionais.

## Telas e APIs

- `/activities` e `GET /api/activities` mostram o catálogo operacional.
- `/demo` e `GET /api/demo/status` mostram a linha do tempo ponta a ponta.
- `/dashboard` e `GET /api/dashboard` mostram KPIs.
- `GET /api/journey/what-to-do-now` retorna a próxima ação baseada na view v1.9.
- `GET /api/validation/e2e/customer-to-billing` valida o fluxo customer-to-billing.

## Como identificar erros

Consulte `logs/scriptcompleto-debug.log`; o `psql` é executado com `ON_ERROR_STOP=1`, preservando mensagem e linha do erro.
