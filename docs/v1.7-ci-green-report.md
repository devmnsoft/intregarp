# v1.7 - CI green report

## Workflows revisados

- `.github/workflows/ci.yml`: restore, build, test, validação de schemas proibidos e typecheck mobile.
- `.github/workflows/database-validation.yml`: PostgreSQL service, aplicação dupla do `scriptcompleto.sql`, objetos críticos e manifesto JSON.
- `.github/workflows/release-candidate.yml`: build de API/Web/Worker, contratos de smoke e Docker compose build quando disponível.

## Como interpretar falhas

- Falha em restore/build/test indica erro real de compilação ou teste.
- Falha em database-validation indica SQL não idempotente, objeto crítico ausente ou schema proibido.
- Falha em release-candidate indica quebra de build por projeto, contrato de smoke ausente ou problema Docker.

## Evidência esperada

Logs dos jobs no GitHub Actions com status verde e resumo no `GITHUB_STEP_SUMMARY`.

## Próxima ação em caso de falha

Corrigir a causa raiz no código, script ou workflow e reexecutar o job. Não mascarar falhas.
