# v1.13 — Limitações conhecidas

Esta fase remove os retornos `501` dos fluxos operacionais principais de clientes, produtos, estoque, pedidos e tarefas e substitui essas respostas por chamadas reais via use cases e repository PostgreSQL.

## Limitações reais ainda existentes

- A validação de build/test não foi concluída neste container porque o SDK do .NET não está instalado.
- A execução real de `database/scriptcompleto.sql` em banco limpo e em segunda execução depende de PostgreSQL/psql fora deste container.
- As ações de comentário de tarefa registram um comentário operacional padrão em `metadata_json`; payload textual dedicado pode ser evoluído sem quebrar o contrato atual.
- A criação de pedido e inclusão de item usam os primeiros cliente/produto ativos do tenant para manter o fluxo E2E executável durante a homologação funcional.

## Não considerados prontos sem validação externa

- Build verde.
- Testes verdes.
- Script completo executado duas vezes.
- Validação SQL v19 executada contra PostgreSQL real.
