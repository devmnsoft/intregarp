# v1.3 Functional Readiness

A v1.3 adiciona checkpoints persistidos, migration versionada, views de validação e API de validação funcional para orientar a substituição dos fluxos críticos in-memory por Dapper/PostgreSQL.

## Critérios cobertos nesta entrega
- Script completo mantido em `/database/scriptcompleto.sql`.
- Migration `0015_v13_funcionalidade_real_end_to_end.sql` criada.
- Views v1.3 de health, telas, pendências, repositories e “O que fazer agora”.
- Controller `FunctionalValidationController` com permissão declarada `validation.functional.visualizar`.
- Testes estáticos para script completo, migration e endpoints de validação.

## Limitações conhecidas
- O ambiente local não possui `dotnet`; build/test .NET devem ser executados em estação/CI com SDK instalado.
- A substituição completa de todos os providers in-memory deve continuar por módulos, sem remover providers fake de sandbox.
