# v1.6 — Release Candidate executável

## Escopo
A v1.6 adiciona validação executável do build, banco, CI, Docker, Windows/IIS, Worker e smoke tests sem substituir o sistema existente. O banco permanece no schema `integrarp` e o `database/scriptcompleto.sql` continua sendo a fonte completa e idempotente.

## Evidência local nesta execução
- `dotnet clean/restore/build/test`: não executado porque o SDK `dotnet` não está instalado no ambiente Codex.
- Mobile: `npm install` e `npm run typecheck` passaram em `apps/IntegraRP.Mobile`.
- Docker: indisponível no ambiente Codex.

## Como validar
1. `dotnet clean && dotnet restore && dotnet build && dotnet test`.
2. `pwsh scripts/db-reset-local.ps1`.
3. `pwsh scripts/db-apply-scriptcompleto.ps1`.
4. `pwsh scripts/validate-docker-release.ps1`.
5. `pwsh scripts/smoke-test-api.ps1` e `pwsh scripts/smoke-test-customer-journey.ps1`.

## Limitações conhecidas
Os endpoints v1.6 informam status `warning` quando a validação depende de PostgreSQL/Docker/IIS reais no ambiente executor.
